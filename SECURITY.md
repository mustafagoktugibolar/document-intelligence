# Security Policy

## Supported Versions

The following versions of this project are currently supported with security updates:

| Version | Supported |
| ------- | --------- |
| 5.1.x   | ✅ Yes     |
| 5.0.x   | ❌ No      |
| 4.0.x   | ✅ Yes     |
| < 4.0   | ❌ No      |

We strongly recommend using the latest supported version to ensure you receive important security patches and updates.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### 📩 How to Report
- Send an email to: **security@yourdomain.com** *(replace with your actual email)*
- Include the following details:
  - A clear description of the vulnerability
  - Steps to reproduce the issue
  - Potential impact and affected components
  - Proof of concept (if available)
  - Suggested fix (optional)

### ⏱ Response Timeline
- **Initial response:** within 48 hours  
- **Status update:** within 3–5 business days  
- **Resolution target:** depends on severity; critical issues are prioritized

### 🔒 Responsible Disclosure
- Please **do not publicly disclose** the vulnerability until it has been resolved
- Avoid accessing, modifying, or deleting data that does not belong to you
- Only test against systems you have permission to assess

### ✅ What to Expect
- If the vulnerability is **accepted**, we will:
  - Investigate and validate the issue
  - Prioritize and implement a fix
  - Notify you when the issue is resolved
  - Optionally credit you in release notes or security advisories

- If the vulnerability is **declined**, we will:
  - Provide a clear explanation (e.g., not reproducible, out of scope, expected behavior)

---

## Scope

This policy applies to:
- Supported versions listed above
- Official releases of this repository

The following are considered out of scope:
- Unsupported or deprecated versions
- Issues caused by third-party dependencies (should be reported to the respective maintainers)
- Misconfigurations in user environments

---

## Security Best Practices

To help keep your deployment secure:
- Always use the latest supported version
- Keep dependencies up to date
- Use secure configuration for secrets and environment variables
- Restrict access to sensitive services and endpoints
- Monitor logs and system activity for unusual behavior
